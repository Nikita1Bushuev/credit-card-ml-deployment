# План A/B-тестирования модели прогнозирования дефолта

## Цель теста
Сравнить текущую production-модель (v1) с новой версией (v2) и принять решение о замене, если v2 демонстрирует статистически значимое улучшение ключевых метрик без ухудшения дополнительных.

## Разделение трафика

Участники тестирования:
- **Контрольная группа (A)**: запросы обслуживаются моделью v1.
- **Тестовая группа (B)**: запросы обслуживаются моделью v2.

**Cпособ разделения трафика** - случайное равномерное распределение входящего трафика (50/50).  Реализуется через параметр `model_version` в запросе (клиент выбирает версию случайным образом на своей стороне).

**Продолжительность теста** - 14 дней, решение о досрочном завершении принимается, если достигнут требуемый уровень значимости.

## Метрики 

### Основная метрика
F1-score для класса дефолт (default=1). Отражает баланс между точностью (Precision) и полнотой (Recall), что критично для задачи кредитного скоринга, где одинаково важны и выявление рисков, и минимизация ложных отказов.

### Дополнительная метрика
Precision для класса дефолт. Показывает долю действительно дефолтных клиентов среди тех, кому модель предсказала дефолт. Высокий Precision снижает риск ошибочного отказа платёжеспособным заявителям, что напрямую влияет на потерю дохода банка.

## Статистический анализ

### Используемый статистический тест
- Для F1-score (непрерывная метрика) применяется **t-критерий Стьюдента для независимых выборок** (или, при большом количестве наблюдений, Z-тест). F1 будет вычисляться на накопленной выборке предсказаний по каждой группе. 
- Для **Precision** (пропорция) используется **Z-тест для разницы двух пропорций**.

### Доверительные интервалы
Все метрики оцениваются с 95% доверительными интервалами (уровень значимости α = 0.05). 
Интервалы строятся:
- Для F1 - на основе t-распределения (или бутстрэп-перцентилей).
- Для Precision - по формуле Уилсона.

### Критерий успешности теста.
Тест успешный, если:
Основная метрика F1-score для v2 статистически значимо выше, чем для v1 (p-value < 0.05), и дополнительная метрика Precision не снижается (p-value для разницы Precision > 0.05, либо нижняя граница доверительного интервала разницы выше допустимого порога). Если значимого улучшения не обнаружено, модель v2 не внедряется.
